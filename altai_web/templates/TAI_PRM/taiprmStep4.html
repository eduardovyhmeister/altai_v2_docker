Some Definitions

This section describes in detail the activity e-risk management process. Three considerations are related to the ISO 31000: (i) the ethical management process encompasses the use of the context provided by the ISO 31000475 and defined in previous steps; (ii) the activities related to the ISO 31000 in communication and consultation are not included in the process but in the RASP architecture; and, finally, (iii) the ethical risk management process contains the ISO-defined Monitoring and Review process defined as a main component after the Risk Evaluation and Risk Treatment. hThe following figure shows a high level diagram of the process

Establishing the context

In this activity, the user must identify the requirements, definitions and interconnections that the AI artifact requires with respect to its usage by humans and other AI artifacts. More specifically, the activity encompasses 485 the following actions:
  • Connectivity with other components and subsystems: Users must define the effects of the AI artifact on the overall system. This is key for cases when AI artifacts are running concurrently on a system to perform system outputs. Additionally, users can understand the effects of the failure modes on the parts where it is connected, improving the accountability and transparency. The connectivity must be established in software architecture documentation.
  • Dependencies: This item refers to the hierarchical extension of the components. Users must track cascade effects on risk analysis, for example using root cause analysis.
  • Constraint and Context: Constraints are set up as requirements to control functionalities, input behaviours, system outcomes, and components values. If relevant, these constraints can be established with the physical context, enhancing the system security especially on AI user interaction.
  • Diagrams: Diagrams should be constructed based on the information collected in Connectivity with other components and subsystems, Dependencies, and Human-AI and Human-UI interactions. Combining these activities with the feedback of previously mentioned contextualization steps, the system security, safety, accountability, and transparency will improve.
